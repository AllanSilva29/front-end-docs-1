import { Aside, Tabs, TabItem } from '@astrojs/starlight/components';

Aplicações React raramente são estáticas. A interface do usuário precisa mudar em resposta a diferentes condições: um usuário está logado ou não? Há itens no carrinho de compras? Os dados de uma API já foram carregados?

**Renderização Condicional** é a prática de exibir diferentes componentes ou elementos JSX com base no estado atual da aplicação. React é flexível e permite que você use o JavaScript que já conhece para isso.

Vamos explorar as técnicas mais comuns.

### Técnica 1: Operador Ternário

O operador condicional ternário (`condição ? exprSeVerdadeira : exprSeFalsa`) é perfeito para uma lógica simples de `if-else` diretamente no seu JSX. É conciso e muito legível para alternar entre duas opções.

<Tabs>
<TabItem label="Exemplo">
```jsx
function LoginStatus({ isLoggedIn }) {
  return (
    <div>
      {isLoggedIn
        ? <p>Bem-vindo(a) de volta!</p>
        : <p>Por favor, faça o login para continuar.</p>
      }
    </div>
  );
}

// Uso:
// <LoginStatus isLoggedIn={true} />  -> Renderiza "Bem-vindo(a) de volta!"
// <LoginStatus isLoggedIn={false} /> -> Renderiza "Por favor, faça o login..."
```
</TabItem>
</Tabs>

### Técnica 2: Operador Lógico `&&` (Curto-circuito)

Quando você quer renderizar algo *somente se* uma condição for verdadeira (e não renderizar nada caso contrário), o operador lógico `&&` é a ferramenta ideal.

Em JavaScript, a expressão `true && expressao` sempre avalia para `expressao`, e `false && expressao` sempre avalia para `false`. O React trata `false`, `null` e `undefined` como "nada a ser renderizado".

<Tabs>
<TabItem label="Exemplo">
```jsx
function Mailbox({ unreadMessages }) {
  const messageCount = unreadMessages.length;

  return (
    <div>
      <h1>Sua Caixa de Entrada</h1>
      {messageCount > 0 &&
        <h2>
          Você tem {messageCount} mensagens não lidas.
        </h2>
      }
    </div>
  );
}

// Uso:
// <Mailbox unreadMessages={['msg1', 'msg2']} /> -> Renderiza o h2
// <Mailbox unreadMessages={[]} />              -> Não renderiza o h2
```
</TabItem>
</Tabs>

<Aside type="caution">
Cuidado ao usar o operador `&&` com números. A expressão `0 && <Component />` renderizará o número `0` na tela, o que geralmente não é o desejado. Para esses casos, é mais seguro converter a condição para um booleano, como `count > 0 && <Component />`.
</Aside>


### Técnica 3: Declarações `if/else` com Variáveis

Para lógicas condicionais mais complexas, usar operadores ternários ou `&&` pode tornar o JSX confuso e difícil de ler. Nesses casos, a melhor abordagem é usar uma declaração `if/else` tradicional fora do `return` para preparar o conteúdo a ser renderizado.

Isso mantém a lógica de programação separada da marcação, resultando em um código mais limpo.

<Tabs>
<TabItem label="Exemplo">
```jsx
import { useState, useEffect } from 'react';

function DataFetcher() {
  const [data, setData] = useState(null);
  const [isLoading, setIsLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    // Lógica para buscar dados de uma API...
  }, []);

  let content;

  if (isLoading) {
    content = <p>Carregando dados...</p>;
  } else if (error) {
    content = <p>Ocorreu um erro: {error.message}</p>;
  } else {
    content = (
      <ul>
        {data.map(item => <li key={item.id}>{item.name}</li>)}
      </ul>
    );
  }

  return (
    <div>
      <h1>Lista de Dados</h1>
      {content}
    </div>
  );
}
```
</TabItem>
</Tabs>

Escolher a técnica certa depende da complexidade da sua lógica. Para decisões simples, o ternário e o `&&` são ótimos. Para cenários mais elaborados, usar `if/else` com uma variável de conteúdo é a abordagem mais robusta e legível.
