Um componente React é dinâmico porque sua aparência pode mudar com o tempo. Essas mudanças são controladas por dois tipos de "dados": **`props`** e **`state`**. Entender a diferença entre eles é a chave para construir aplicações interativas.

A maneira mais simples de pensar sobre isso é:
- **`props`** são como os argumentos de uma função.
- **`state`** é como a memória interna de uma função, que ela pode lembrar e modificar.

## Props: Passando Dados de Cima para Baixo

**`props`** (abreviação de "properties") são a forma como um componente pai passa dados para seus componentes filhos. Elas permitem que você personalize e configure os componentes.

<Card title="Princípio Chave: Props são Imutáveis">
  Uma das regras mais importantes do React é que um componente **nunca deve modificar as `props` que recebe**. Elas são somente leitura (read-only). Essa restrição garante um fluxo de dados previsível, conhecido como **fluxo de dados unidirecional**.
</Card>

**Exemplo:** Um componente `UserProfile` recebe informações do usuário via `props`.

```jsx
// Componente Filho: UserProfile
// Ele recebe 'name' e 'age' e apenas os exibe.
function UserProfile(props) {
  return (
    <div className="profile">
      <p>Nome: {props.name}</p>
      <p>Idade: {props.age}</p>
    </div>
  );
}

// Componente Pai: App
// Ele "possui" os dados e os passa para o filho.
function App() {
  return <UserProfile name="Carlos" age={30} />;
}
```

## State: A Memória do Componente

**`state`** (estado) é um conjunto de dados que é gerenciado **dentro** de um componente. Ele representa informações que podem mudar ao longo do tempo, geralmente em resposta a interações do usuário.

Quando o `state` de um componente é atualizado, o React automaticamente **re-renderiza** o componente para refletir a nova informação na tela.

Para usar o `state` em um componente de função, usamos o Hook `useState`.

**Exemplo:** Um componente de contador que rastreia o número de cliques.

```jsx
import { useState } from 'react';

function Counter() {
  // 1. Declaramos uma variável de estado 'count' com valor inicial 0.
  //    'setCount' é a função que usamos para atualizar o 'count'.
  const [count, setCount] = useState(0);

  // 2. Função que atualiza o estado quando o botão é clicado.
  const handleClick = () => {
    setCount(count + 1);
  };

  return (
    <div>
      <p>Você clicou {count} vezes.</p>
      <button onClick={handleClick}>
        Clique aqui
      </button>
    </div>
  );
}
```

Neste exemplo, `count` é o `state` do componente `Counter`. Ele é privado e controlado inteiramente pelo próprio componente.

## Tabela Comparativa: Props vs. State

| Característica | Props                                       | State                                         |
| -------------- | ------------------------------------------- | --------------------------------------------- |
| **Origem**     | Passadas de um componente pai.              | Gerenciado dentro do próprio componente.      |
| **Controle**   | Controladas pelo componente pai.            | Controlado e privado ao próprio componente.   |
| **Mutabilidade**| Imutáveis (somente leitura) para o filho.   | Mutável (usando a função `setState`).         |
| **Propósito**  | Configurar e passar dados para um componente. | Gerenciar dados que mudam com o tempo.        |
| **Fluxo**      | Fluxo de dados de cima para baixo.          | Encapsulado, aciona re-renderizações locais.  |

<Aside type="note">
Uma boa regra geral: se um componente precisa "lembrar" de algo que muda com a interação do usuário (como um valor de input, se um modal está aberto, etc.), essa informação pertence ao **`state`**. Se um componente apenas precisa exibir informações que lhe são dadas, ele usa **`props`**.
</Aside>
